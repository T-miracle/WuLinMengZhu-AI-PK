# 武林萌主战斗模拟器

根据项目内的装备表和规则说明实现的 Rust 蒙特卡洛模拟器。

## 运行

使用内置示例：

```powershell
cargo run --release -- --runs 2400
```

使用自己的输入文件：

```powershell
cargo run --release -- input.txt --runs 2400 --seed 20260706
```

也可以通过 stdin 输入；格式支持项目需求中这种 5 行 4 列文本表格。装备名后可以追加 `1`/`2`/`3`/`4` 表示星级，例如 `冰魄2`、`龙弧4`、`烛神令3`；未追加数字时沿用该装备文件里的默认基础值。

输出格式包含：

- 程序模拟结论和胜场表。
- 核心时间线，按双方关键命中节点汇总平均伤害、双方剩余血量和击杀结果，并以 Markdown 表格输出。

## 代码结构

- `src/main.rs`：模拟主流程、输入解析、战斗循环、战报输出。
- `src/equipment/mod.rs`：装备注册入口和通用效果枚举。
- `src/equipment/items/`：每件装备一个文件，包含该装备的基础属性、星级属性和效果枚举。

每个装备文件包含两层信息：

- `DEF`：名称、类型、占格、基础攻击、发动间隔。
- `PROPERTIES`：由 `Trigger` 触发时机和 `Effect` 原子效果组合而成，例如 `Damage`、`Heal`、`Freeze`、`Accelerate`、`Charge`、`AttackBonus`、`Sword` 等。
- `STAR_PROFILES`：可选星级属性表，用于记录 1/2/3/4 星的攻击力、发动间隔，以及会随星级变化的剑势或联动数值。

## 当前规则假设

- 默认生命 1000，战斗上限 20 秒，模拟步长 0.05 秒。
- 冻结、加速、减速、充能作用于有读条的装备：近战武器和护符。
- 加速期间读条速度 +100%，减速期间读条速度 -50%，冻结期间读条暂停。
- 蓄力攻击发动时，若防守方剑势 >= 50，则防守方消耗 50 剑势振刀，攻击方硬直 3 秒，本次蓄力无效。
- 护甲先吸收伤害，剩余伤害再扣生命；减伤先对总伤害生效。
- 战斗结束后，0~20 秒时间线会用最终血量补齐，事件不再继续发生。
- 随机目标从可被冻结/读条的装备中抽取；同一次多目标冻结不会重复抽同一件。

装备基础属性和效果说明集中在 `src/equipment/items/`；战斗执行细节仍在 `src/main.rs` 中调度。如果后续确认官方细节不同，可以优先调整对应装备文件，再同步主流程中的执行逻辑。
