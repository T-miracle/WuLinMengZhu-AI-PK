# 武林萌主战斗模拟器

根据项目内的装备表和规则说明实现的 Rust 蒙特卡洛模拟器。

## 运行

使用内置示例：

```powershell
cargo run --release -- --runs 2400
```

使用自己的输入文件：

```powershell
cargo run --release -- input.txt --runs 2400 --seed 20260706
```

也可以通过 stdin 输入；格式支持项目需求中这种 5 行 4 列文本表格。装备名后可以追加 `1`/`2`/`3`/`4` 表示星级，例如 `冰魄2`、`龙弧4`、`烛神令3`；目前只有 `冰魄`、`龙弧`、`烛神令` 支持独立星级档位，其他未分档装备默认按 `4` 星处理。

## 代码结构

- `src/main.rs`：模拟主流程、输入解析、战斗循环、战报输出。
- `src/equipment/mod.rs`：装备数据库初始化和 SQLite 查询入口。
- `src/equipment/catalog.rs`：装备基础数据和星级档位种子；程序启动时会自动同步到本地 `equipment.sqlite`。

装备数据库目前包含四类核心数据：

- `equipment`：名称、程序类型、显示类型、占格、默认星级、基础攻击、发动间隔、完整属性描述、关联术语文本。
- `equipment_star_profiles`：可选星级属性表，用于记录 1/2/3/4 星的攻击力、发动间隔，以及会随星级变化的剑势或联动数值。
- `attribute_terms`：术语说明表，对应附属属性说明。
- `equipment_attribute_terms`：装备与术语的关联表，便于筛选和人工维护。

程序启动时会补齐缺失的表、字段和默认种子数据，但不会覆盖数据库里已经存在的非空人工维护内容。

## 当前规则假设

- 默认生命 1000，战斗上限 20 秒，模拟步长 0.05 秒。
- 冻结、加速、减速、充能作用于有读条的装备：近战武器和护符。
- 加速期间读条速度 +100%，减速期间读条速度 -50%，冻结期间读条暂停。
- 蓄力攻击发动时，若防守方剑势 >= 50，则防守方消耗 50 剑势振刀，攻击方硬直 3 秒，本次蓄力无效。
- 暴击按攻击方暴击率随机触发，触发后本次伤害变为 2 倍。
- 灼伤每秒造成等同于当前灼伤层数的伤害，并在每次结算后让层数降低 10%。
- 中毒每秒造成等同于当前中毒层数的伤害，并无视护甲。
- 护甲先吸收伤害，剩余伤害再扣生命；减伤先对总伤害生效，但不影响灼伤和中毒伤害。
- 战斗结束后，0~20 秒时间线会用最终血量补齐，事件不再继续发生。
- 随机目标从可被冻结/读条的装备中抽取；同一次多目标冻结不会重复抽同一件。

装备基础属性和星级档位由 `src/equipment/catalog.rs` 自动同步到本地 `equipment.sqlite`；战斗执行细节仍在 `src/main.rs` 中调度。如果后续确认官方细节不同，可以优先调整 catalog 数据，再同步主流程中的执行逻辑。
